# STORY-260224-27zlty: makefile-automation

## Description
Makefile as THE SINGLE entry point for ALL project operations. No one calls tuist/xcodebuild/swiftlint/swiftformat directly — everything goes through make targets.

Core targets:
- make setup — full project setup from scratch (install tools, fetch deps, tuist generate)
- make resetup — nuke everything + setup from scratch
- make generate — tuist generate
- make build — xcodebuild build
- make test — run all tests
- make clean — clean derived data, build artifacts
- make lint — swiftlint or whatever linter
- make format — swiftformat or whatever formatter
- make validate — run all checks (lint + build + test)
- make install-tools — install tuist, linters, formatters, etc

Principle: users and agents NEVER call tuist, xcodebuild, swiftlint, swiftformat directly. Makefile abstracts all tooling. If the underlying tool changes (e.g. swiftlint → periphery), only the Makefile target changes — callers dont care.

Makefile is generated by ios-app-manager init, templated from project config (knows app name, schemes, targets). Extensible: users add custom targets alongside generated ones.
