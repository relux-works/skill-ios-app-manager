# Module Audit: TokenProvider

## Package
`internal/tokenprovider/`

## CLI Command
`token-provider setup` — defined in `internal/cli/token_provider.go`

---

## 1. Setup() Flow (step by step)

1. **Validate input** — requires non-empty `ProjectRoot` and `AppName`
2. **Resolve platform** — defaults to `iOS(.v17)` if not provided
3. **Resolve modules path** — via `ioc.ResolveModulesPath()` (defaults to `Packages/`)
4. **Create interface package** `TokenProvider/`:
   - Creates directory `<ModulesPath>/TokenProvider/`
   - Generates `Package.swift` via `tuistproj.GeneratePackageSwift()` (type: Interface)
   - Creates `Sources/TokenProvider/` directory
   - Writes `.module-type` marker file with value `kit`
5. **Create impl package** `TokenProviderImpl/`:
   - Creates directory `<ModulesPath>/TokenProviderImpl/`
   - Generates `Package.swift` via `tuistproj.GeneratePackageSwift()` (type: Impl)
   - Creates `Sources/TokenProviderImpl/` directory
6. **Render Swift files into interface package:**
   - `Sources/TokenProvider/TokenProvider.swift` — namespace enum (inline, not template)
   - `Sources/TokenProvider/Module/TokenProvider.Module.swift` — module declaration (inline)
   - `Sources/TokenProvider/Module/TokenProvider.Module+Interface.swift` — protocol (from template)
   - `Sources/TokenProvider/TokenProvider.AuthData.swift` — auth data struct (from template)
7. **Render Swift files into impl package:**
   - `Sources/TokenProviderImpl/Module/TokenProvider.Module+Impl.swift` — actor implementation (from template)
8. **Patch Project.swift** — adds `.external(name: "TokenProvider")` and `.external(name: "TokenProviderImpl")` to dependencies
9. **Patch root Package.swift** — adds `.package(path: "Packages/TokenProvider")` and `.package(path: "Packages/TokenProviderImpl")`
10. **Patch Workspace.swift** — adds same package references (skips if file doesn't exist)
11. **Regenerate Registry.swift** — if IoC is set up (Registry.swift exists):
    - Discovers all modules via `ioc.DiscoverModules()`
    - Preserves `HasRelux` flag by checking existing Registry.swift for Relux imports
    - Calls `ioc.ScaffoldRegistryWithData()` to regenerate

## 4. SetupInput Fields

```go
type SetupInput struct {
    ProjectRoot string  // required — project root directory
    AppName     string  // required — from config, used for Registry.swift path resolution
    ModulesPath string  // optional — defaults to "Packages" via ioc.ResolveModulesPath()
    Platform    string  // optional — SwiftPM platform, defaults to "iOS(.v17)"
}
```

**From CLI command** (`internal/cli/token_provider.go`):
- `ProjectRoot` = `filepath.Dir(selectedConfigPath)` (derived from config file location)
- `AppName` = `cfg.AppName` (from config JSON)
- `ModulesPath` = `cfg.ModulesPath` (from config JSON, normalized)
- `Platform` = **not set from CLI** (always uses default `iOS(.v17)`)

---

## 5. Prerequisites

- **Project.swift** must exist with a `dependencies:` array
- **Package.swift** (root) must exist with a `dependencies:` array
- **IoC setup** (optional but recommended) — if `Registry.swift` exists at `Targets/<AppName>/Sources/App/<AppTypeName>.Registry.swift`, it gets regenerated. Without IoC setup, registry patching is silently skipped.
- **No hard dependency on other modules** — tokenprovider does not import or reference SecureStore, Relux, or any other module. It is a standalone foundation module.

**None.** The CLI command has no custom flags beyond the global `--config` option. Platform is hardcoded to default.

**foundation** — module type marker is `kit`, which maps to the `foundation` category in Registry.swift.

## 8. Registry Module Struct Fit Assessment

**Fits cleanly.** This module is a straightforward candidate for the registry pattern:

- **ID**: `"token-provider"`
- **Name**: `"TokenProvider"`
- **Dependencies**: `[]` (no module prerequisites; IoC is optional)
- **Category**: `"foundation"` (kit -> foundation mapping)
- **Setup**: Direct mapping to `tokenprovider.Setup(input)` — input fields align with common `SetupInput`
- **Plan**: Can describe "Creates TokenProvider + TokenProviderImpl packages with protocol/actor pattern"
- **UsageGuide**: "TokenProvider setup complete" (current message)

**No special needs:**
- No custom flags (unlike SecureStore's `--access-group`)
- No external Swift package dependencies to add (unlike relux-feature's `swift-relux`)
- Standard interface/impl split pattern
- Idempotent — safe to run multiple times
- `.module-type` marker already written (`kit`)

**One minor note:** The `Platform` field in `SetupInput` exists but is never set from CLI (always defaults). The unified `SetupInput` could either include it as optional or drop it.

---

## Key Takeaways

1. TokenProvider is one of the simplest modules — no flags, no external deps, no prerequisites
2. Clean interface/impl split with actor-based implementation (Swift concurrency friendly)
3. AuthData is a simple value type (accessToken, refreshToken, acquireDate, ttl)
4. Module type `kit` -> maps to `foundation` category in Registry
5. Registry regeneration is conditional — only if IoC was previously set up
6. All manifest patching (Project.swift, Package.swift, Workspace.swift) is idempotent
7. Perfect fit for the unified Module struct — no adaptation needed
